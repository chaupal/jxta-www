Developer Spotlight



Philippe Moulin is currently working on a project called net2map. He works on Project JXTA about 50% of his work time. On the mailing lists he is known as moulinp1. He lives in Paris, France.

Net2map is a multidimensional distributed search engine. A "dimension" is anything that can translate a document into a value: User rating, category, author, price (if the document describes an item that is for sale), etc. Currently, only three simple dimensions are available: creation date, size, and relevancy (provided by Lucene, when a keyword search is done). More dimensions are planned to make it easy for anyone to add their own dimensions to the system.

Search results are displayed on a map. Each result is a small button whose horizontal and vertical position matches it's dimensions values. So, when you make a query on net2map, it's like looking at a particular location of this "map of the net".

When originally evaluating technologies for n2map, Philippe selected Project JXTA because there really weren't any other open-source P2P frameworks available. As net2map needs to have many simultaneous users to become useful, he was looking for a P2P technology that would be usable anywhere, by anyone. He was specifically looking at:

  • Open-source licensing: net2map is GPL licensed
  • Usable on any computer, including MS Windows machines.
  • Very easy to install and use: Opening a web-based search engine, is faster and easier than installing some software. Using Java
    web start, users will be able to try net2map almost as easily as they use a web-based search engine.
  • Automated software updates; net2map releases a new version almost every day!.

Net2map is currently available only in French. Philippe is working on translating the site to English prior to proposing the project at jxta.org.

Looking back, Philippe believes that there was an advantage to selecting JXTA technology. With all that is available from Project JXTA, he did not need to rewrite a new, proprietary, and incompatible-with-the-rest-of-the-world P2P framework. Also, by selecting Project JXTA, he was joining a cadre of software engineers who provided their knowledge and experiences when he needed assistance.

Looking to the future, he hopes that P2P will spread from computers to... people! He explains:

"In the client-server world, client computers are inherently passive and isolated from each other, and so are their users. It is like being in a mostly-silent crowd, on a street: you may move, visit some shops, watch street performers ... but you don't see other "users" ... except when they are in your way. And ... wait, my comparison is flawed: you can't just "walk" in a client-server network, if something is not listed anywhere, it just don't exist.

The P2P world may be a lot noisier, because in a P2P crowd anyone may talk to anyone, but it is a lot more interesting. It just needs a very advanced search facility.

Mail, newspapers, and shops were there before the Internet. I believe that, with technologies like project JXTA, we can build something REALLY new, not just improved versions of old things."
