Get involved

As with any open source project, our goal is to get the community involved by contributing to JXTA technology. There are many ways of contributing to an open source software project. Here are some ways that you can immediately get involved:

  • File a bug report - Detailed bug reports are valuable to developers. Next time you come across something you think is a bug, take some time to reproduce it. If you can, file a bug report by cutting and pasting the relevant information from your log file, including any stack traces and uncaught exceptions. You can file bug reports using Issuezilla, the JXTA bug-tracking system.
  • Join a mailing list - Join the mailing lists to post general feedback, feature requests, and requests for help. Also, if you see a question on the list that you can answer, get to it! See the mailing lists page for details on how to subscribe.
  • Contribute code - In accordance with the JXTA governance process, you can submit code to the project leader as a contribution either as a user, a developer, or as a committer. If your code is accepted to be integrated into the source tree, then you will be asked to sign a Contributor's Agreement similar to what Apache Software Foundation requires.
  • Start a new project - JXTA Community Members can start new projects by following the JXTA Community Project Proposal Guidelines.

Take a look and join our efforts. We welcome your feedback, comments, and code.

For more information, view JXTA whitepapers, Project Proposal Guide and Frequently Asked Questions.
